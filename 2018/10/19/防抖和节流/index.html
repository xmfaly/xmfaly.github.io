防抖和节流都是为了限制函数的执行频次，达到节约资源的目的。

  • 防抖：任务频繁触发的情况下，只有触发任务的时间超过一定时间间隔才会执行。
  • 节流：在指定的时间间隔内执行一次

节流(throttle)

以监听鼠标运动为例

function move() {
    console.log('move');
}
window.onmousemove = move;

这样不断的执行很浪费性能，节流函数就可以做到在一定的时间间隔内只执行一次，而不需要不间断的去执行函数。

function move() {
    console.log('move');
}
window.onmousemove = throttle(move);

加上节流函数之后，只有把很多连续的操作压缩了，减少了资源的浪费
throttle的实现如下

function throttle(func, time = 300) {
    let canRun = true;
    return function () {
        if (!canRun) {
            return;
        }
        canRun = false;
        setTimeout(() => {
            canRun = true;
            func();
        }, time);
    }
}

通过canRun这个闭包函数标记来实现整体的效果，在函数执行之前判断canRun的状态，如果是false就不继续执行，直到第一次设置的定时器把canRun设置为true才会进行下一个执行周期。

防抖(debounce)

感觉生活中的电梯和防抖有些类似，假如电梯的门是有人通过之后10s关门，那么在这10s中假如又通过了一个人，他会重置这个时间，直到某次10s都没有人在通过门，他才会关门。

假如有一个输入框，这个输入框要和后台交互，为了优化体验，在这个输入框改变的时候就进行交互，而不是失去焦点在进行交互。

// html
<input type="text" oninput="change(event)" />

// js
function change(e) {
    fetch('/test');
}

先看一下没有处理之前的效果

每改变一次就会触发事件，浪费资源。
处理之后

// html
<input type="text" id="input" oninput="_change()" />
// js
function change(e) {
    fetch('/test');
}
const _change = debounce(change);

效果如下

function debounce(func, time = 1000) {
    let timer;
    return function () {
        timer && clearTimeout(timer);
        timer = setTimeout(() => {
            func.apply(this);
        }, time);
    }
}

当然这两个函数都是最简单的实现，不一定可以直接套用，比如某些带参数的函数，某些涉及到作用域的函数都不能用上面的实现方式直接调用，改天没事的时候可以实现一下更加通用的方法。
